##  项目概览

###  1. 角色获取系统（抽卡）

设计目标：在“不确定性”与“玩家预期”之间建立平衡  

**核心机制：**
- 基础概率 + 软保底 + 硬保底 + UP机制  
- 情绪曲线设计（70–90抽为核心出卡区间）  
- Python模拟验证（500+次）  

**设计价值：**
- 降低极端非酋体验  
- 提升玩家预期稳定性  
- 为战斗系统提供稳定角色供给  

---

###  2. 战斗系统设计（宠物Demo）

以轻策略为核心，构建阵容搭配与战斗节奏  

**核心循环：**
宠物 → 养成 → 阵容 → 战斗 → 反馈  

**系统设计：**
- 定位：输出 / 防御 / 控制  
- 属性：ATK / DEF / SPD  
- 战斗：回合制 + 速度顺序 + 技能CD  

**技能示例：**
- 潮汐冲击  
  - 120%伤害  
  - 降速10%（2回合）  

**设计目标：**
- 强化阵容策略选择  
- 通过技能组合控制战斗节奏  
- 通过数值设计保证平衡性  

---

###  3. 养成系统（资源循环）

将重复行为转化为“决策驱动循环”  

**核心循环：**
资源 → 转化 → 强化 → 用于战斗  

**设计亮点：**
- 多路径资源选择  
- 即时收益 vs 长期收益权衡  
- 每日限制 + 概率奖励控制节奏  

**设计价值：**
- 提升长期参与动力  
- 支撑战斗成长系统  
- 避免资源与数值失衡  

---

##  数据验证

- 蒙特卡洛模拟（500+次）  
- 出货集中区间：70–90抽  
- 有效降低尾部风险  
- 在保证体验的同时控制付费深度  

---

##  设计思路

- 从玩家体验出发  
- 用系统结构实现  
- 用数据验证设计  

> 系统不仅要可运行，更要能够引导玩家行为
